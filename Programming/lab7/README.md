Лабораторная работа №7: Многопоточное серверное приложение с СУБД

Стек: Java Core, Concurrency (Thread Pools, ForkJoinPool), SQL (PostgreSQL), JDBC, Security (SHA-1).

Что реализовано:

Персистентность данных: Хранение коллекции перенесено из файлов в реляционную СУБД PostgreSQL. Работа с базой организована через JDBC с использованием сиквенсов (sequences) для генерации ID.

Многопоточность: Реализована асинхронная обработка запросов на сервере:

Чтение запросов: Cached thread pool.

Обработка бизнес-логики: ForkJoinPool.

Отправка ответов: ForkJoinPool.

Безопасность (Auth): Система регистрации и авторизации пользователей. Реализовано хэширование паролей алгоритмом SHA-1 для безопасного хранения в БД.

Разграничение прав доступа: Реализована логика владения объектами. Пользователи могут просматривать всю коллекцию, но редактировать или удалять — только созданные ими элементы.

Потокобезопасность: Синхронизация доступа к коллекции в памяти с помощью Collections.synchronizedList.

Надежность: Обновление коллекции в оперативной памяти происходит только после успешной транзакции в базе данных.
