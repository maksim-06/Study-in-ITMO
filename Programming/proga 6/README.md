Лабораторная работа №6: Разработка клиент-серверного приложения

Стек: Java Core, Network (UDP), NIO (DatagramChannel), Serialization, Stream API.

Что реализовано:

Клиент-серверная архитектура: Разделение монолитного приложения на независимые модули. Клиент отвечает за интерактивный ввод и валидацию, сервер — за бизнес-логику и хранение данных.

Сетевое взаимодействие:

Использование протокола UDP для обмена данными.

Реализация серверной части на датаграммах и клиентской на DatagramChannel.

Настройка сетевых каналов в неблокирующем режиме.

Обмен данными: Передача команд и аргументов в виде сериализованных объектов (Object Serialization), что гарантирует строгую типизацию протокола.

Функциональное программирование: Обработка коллекции на сервере полностью переведена на Stream API с использованием лямбда-выражений.

Отказоустойчивость: Реализован механизм обработки временной недоступности сервера на стороне клиента.

Модульность: Сервер спроектирован из независимых модулей приема, чтения, обработки и отправки данных.
