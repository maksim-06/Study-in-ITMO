Лабораторные работы по программированию (Java Enterprise \& CLI)

В этом разделе собраны мои практические работы, выполненные в рамках курса «Программирование» в ИТМО. Проекты демонстрируют последовательное изучение Java: от базового синтаксиса до разработки распределенных многопоточных систем с базой данных.

🚀 Ключевые навыки:

Backend: Java Core (17+), Stream API, Multithreading (JUC), JDBC.

Network: Сетевое взаимодействие по протоколу UDP, использование NIO (DatagramChannel), Сериализация объектов.

Database: Проектирование схем в PostgreSQL, управление транзакциями, безопасное хранение паролей (SHA-1).

Architecture: Применение принципов SOLID, использование абстрактных классов, интерфейсов и современных структур (Records, Enums).

📂 Список проектов:

Лабораторная работа №1: Основы Java и Math API

Технологии: Java SE, форматированный вывод, статические методы.

Результат: Реализация сложных математических вычислений и алгоритмов обработки двумерных массивов.

Лабораторная работа №2: Объектно-ориентированное программирование

Технологии: Наследование, Полиморфизм, Инкапсуляция.

Результат: Моделирование игровой логики на базе внешних библиотек (Pokemon.jar).

Лабораторная работа №3-4: Проектирование систем

Технологии: SOLID, Exceptions, Records, Collections.

Результат: Создание расширяемой объектной модели с вариативными сценариями взаимодействия и кастомной обработкой исключений.

Лабораторная работа №5: Интерактивный CLI-сервис

Технологии: Stack, Buffered Streams, CSV I/O, Валидация данных.

Результат: Консольное приложение для управления коллекцией объектов с поддержкой скриптов и фильтрации.

Лабораторная работа №6: Клиент-серверная архитектура

Технологии: UDP, NIO (неблокирующие каналы), Serialization, Stream API.

Результат: Распределенное приложение с разделением на клиентский (валидация/ввод) и серверный (бизнес-логика) модули.

Лабораторная работа №7: Многопоточный сервер и СУБД

Технологии: PostgreSQL, Thread Pools (Cached, ForkJoin), User Auth (SHA-1), JDBC.

Результат: Полноценная многопоточная система с авторизацией пользователей, разграничением прав доступа и хранением данных в реляционной базе.

🛠 Инструментарий:

Сборка: Gradle / Maven / Ручная сборка JAR.

Среда: IntelliJ IDEA, Linux (сервер Helios).
