Лабораторная работа №1: Проектирование реляционной модели данных

Стек: PostgreSQL, ER-диаграммы (инфологическая модель).

Что реализовано:

— Анализ предметной области: Проведен разбор текстового описания для выделения ключевых сущностей, их атрибутов и связей.

— Инфологическое моделирование: Построена концептуальная схема данных, определены типы связей (one-to-many, many-to-many).

— Даталогическое моделирование: Спроектирована физическая схема базы данных с использованием типов данных PostgreSQL.

— Целостность данных: Реализованы ограничения (Constraints):

\* PRIMARY KEY и FOREIGN KEY для поддержки связей.

\* NOT NULL для обязательных полей.

\* CHECK для контроля бизнес-логики (например, проверка "критической массы").

— Реализация: Написаны SQL-скрипты для создания таблиц (CREATE TABLE) и заполнения их тестовыми данными (INSERT).
