Базы данных (PostgreSQL)

В этом разделе собраны лабораторные работы, охватывающие полный цикл работы с реляционными СУБД: от анализа предметной области и проектирования схем до написания серверной логики и оптимизации запросов.

🚀 Ключевые навыки:

Моделирование: Проектирование инфологических (ER) и даталогических моделей данных.

Проектирование: Нормализация отношений до уровней 3NF и BCNF.

SQL (DML/DDL): Написание сложных аналитических запросов с использованием оконных функций, подзапросов и всех видов JOIN.

Программирование (PL/pgSQL): Разработка триггеров и хранимых процедур для автоматизации бизнес-логики.

Оптимизация: Анализ планов выполнения запросов (EXPLAIN ANALYZE) и проектирование индексов.

📂 Обзор работ:



Лабораторная работа №1: Проектирование реляционной модели

Суть: Трансформация неструктурированного описания предметной области в строгую схему данных.

Результат: Инфологическая модель, создание таблиц в PostgreSQL с учетом ограничений целостности (CHECK, UNIQUE, FOREIGN KEY).



Лабораторная работа №2: Сложные SQL-запросы

Суть: Работа с данными реальной информационной системы (студенты, приказы, ведомости).

Результат: Реализация выборок со множественными соединениями, расчет средних баллов через агрегацию и фильтрация данных через вложенные запросы.



Лабораторная работа №3: Нормализация и PL/pgSQL

Суть: Приведение БД к нормальной форме Бойса-Кодда и внедрение серверной логики.

Результат: Обоснование декомпозиции таблиц, разработка триггеров и функций на языке PL/pgSQL для контроля целостности данных.



Лабораторная работа №4: Оптимизация и индексация

Суть: Анализ производительности запросов на больших объемах данных.

Результат: Использование EXPLAIN ANALYZE для поиска узких мест, внедрение B-tree индексов и сравнение планов выполнения (Cost-based optimization).
