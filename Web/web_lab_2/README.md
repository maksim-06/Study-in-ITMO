Лабораторная работа №2: Сервлеты, JSP и паттерн MVC

Стек: Java Enterprise (Servlets), JSP, JavaScript, CSS, WildFly (App Server).

Что реализовано:

— Архитектура MVC: Четкое разделение ответственности.

\* Controller: ControllerServlet — единая точка входа, которая маршрутизирует запросы.

\* Model: Логика проверки попадания точки в область.

\* View: JSP-страницы для отображения интерфейса и результатов.

— Интерактивный UI: Реализован кликабельный график (SVG/Canvas). При клике JavaScript вычисляет координаты относительно радиуса R и отправляет их на сервер.

— Управление состоянием: Хранение истории проверок в рамках HTTP-сессии или Bean-компонента (данные не пропадают при обновлении страницы).

— Валидация: Двухуровневая проверка данных (на стороне клиента через JS и на стороне сервера в сервлетах).

— Deployment: Приложение развернуто на сервере приложений WildFly в standalone-конфигурации.
