📂 СТРУКТУРА ФАЙЛОВ В ПАПКЕ



├── 📄 build.xml             # Главный скрипт автоматизации Apache Ant

├── 📄 build.properties      # Базовые переменные и пути для стандартной сборки

├── 📄 alt.properties        # Параметры для альтернативной сборки (цель alt)

├── 📄 env.properties        # Настройки окружения JVM (цель env)

└── 📄 watch.properties      # Конфигурация со списками отслеживаемых классов и параметров



\*Примечание: Исходный код самого Java-приложения подключается в конфигурации внешне через относительные пути к лабораторной работе по Веб-программированию.\*



🛠 СПИСОК РЕАЛИЗОВАННЫХ ЦЕЛЕЙ (ANT TARGETS)



💎 compile — компиляция исходных кодов веб-проекта.

💎 build — компиляция и упаковка проекта в исполняемый jar-архив с автоматической генерацией MANIFEST.MF (вызывает compile).

💎 clean — полное удаление скомпилированных классов и всех временных файлов сборки.

💎 test — автоматический запуск junit-тестов (перед стартом обязательно вызывает build).

💎 native2ascii — кодирование и извлечение строковых параметров из классов в файлы локализации.

💎 xml — автоматическая валидация всех xml-файлов конфигурации в структуре проекта.

💎 scp — перемещение собранного jar-архива по протоколу SCP на удаленный server (требует build).

💎 music — воспроизведение звукового сигнала / аудиофайла по завершении успешной сборки (требует build).

💎 doc — автоматическая генерация документации javadoc и добавление хэш-сумм MD5 и SHA-1 в MANIFEST.MF.

💎 team — выгрузка из svn-репозитория 3 предыдущих ревизий, их сборка и архивация в zip (использует build).

💎 report — сохранение xml-отчета junit в репозиторий svn и автоматический коммит при успешных тестах.

💎 alt — создание альтернативной версии программы с заменой имен переменных/классов и упаковка в jar (использует build).

💎 history — автоматический откат по истории svn до последней рабочей ревизии при ошибках компиляции и генерация diff-файла.

💎 env — сборка и запуск приложения в альтернативных окружениях с подменой аргументов JVM.

💎 diff — проверка состояния рабочей копии и автоматический commit в репозиторий git, если изменения коснулись классов из файла параметров.
