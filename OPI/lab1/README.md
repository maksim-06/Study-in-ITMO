🛠 ЧТО РЕАЛИЗОВАНО В ХОДЕ РАБОТЫ



💎 Спецификация требований (SRS): Разработан подробный документ в соответствии с методологией RUP с классификацией всех типов требований.

💎 Пользовательские требования: Описан функционал поиска путеводителей, публикации вопросов/ответов и ведения личных журналов путешествий.

💎 Бизнес-требования: Спроектированы инструменты монетизации портала, модерации контента и аналитики пользовательской активности для владельцев.

💎 Нефункциональные требования: Зафиксированы жесткие параметры производительности, масштабируемости, безопасности и кроссбраузерности системы.

💎 Атрибутивный анализ: Для каждого требования индивидуально определены текущий статус, приоритет, риски и трудозатраты в человеко-часах.

💎 Визуализация (UML): Разработаны Use-Case диаграммы, описывающие сценарии взаимодействия акторов (Пользователь, Редактор, Администратор) с порталом.



💻 ТЕХНОЛОГИЧЕСКИЙ СТЕК



📝 Методология: RUP (Rational Unified Process)

📐 Стандарт документации: SRS (Software Requirements Specification)

📊 Инструменты моделирования: UML Use-Case Diagrams
