Лабораторная работа №1: Анализ требований и проектирование системы (SRS \& UML)

Проект: Веб-портал для путешественников (на базе TravelAsk).

Что реализовано:

Спецификация требований (SRS): Разработан документ в соответствии с методологией RUP. Все требования классифицированы и детально описаны.

Функциональные требования:

Пользовательские: Поиск путеводителей, публикация вопросов/ответов, ведение журналов путешествий.

Бизнес-требования (владельцы): Инструменты монетизации, модерация контента, аналитика пользовательской активности.

Нефункциональные требования: Описаны параметры производительности, масштабируемости, безопасности и кроссбраузерности системы.

Атрибутивный анализ: Для каждого требования указаны статус, приоритет, риск и трудозатраты (в человеко-часах).

Визуализация (UML): Спроектированы Use-Case диаграммы, описывающие основные сценарии взаимодействия акторов (пользователь, редактор, админ) с системой.
