Основы программной инженерии (Software Engineering)

Этот раздел посвящен изучению промышленных стандартов разработки, жизненного цикла ПО и инструментов обеспечения качества. Курс находится в процессе прохождения (Spring 2024).

🚀 Текущие компетенции:

System Analysis: Сбор и классификация требований (функциональные/нефункциональные), работа с методологией RUP.

UML Modeling: Проектирование архитектуры системы через диаграммы прецедентов (Use Case).

Version Control: Уверенная работа с Git (ветвление, слияние, разрешение конфликтов) и понимание работы SVN.

Estimation: Аргументированная оценка трудозатрат на реализацию программных модулей.

📂 Обзор выполненных работ:

Лабораторная работа №1: Анализ требований и проектирование (SRS \& UML)

Проект: Разработка спецификации для веб-портала путешественников.

Результат: Документ SRS по стандарту RUP и спроектированные Use-Case диаграммы. Оценка приоритетов и рисков разработки.



Лабораторная работа №2: Управление версиями в командной разработке

Задача: Воспроизведение сложной топологии коммитов и веток в Git и SVN.

Результат: Отработка навыков синхронизации изменений и ручного разрешения конфликтов при параллельной разработке.
