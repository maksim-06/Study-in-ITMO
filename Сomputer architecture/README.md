🎯 ЦЕЛЬ КУРСА



Главная задача курса — полностью устранить "магическое мышление" и сформировать у инженера масштабный кругозор, позволяющий смотреть на системы не только с программистской, но и с аппаратной точки зрения. 



Курс дает базу, чтобы разработчик умел делать правильные предположения о том:

💎 Как устроены системы, с которыми он работает (сервисы, библиотеки, фреймворки, процессоры).

💎 Как должны быть спроектированы системы, которые он создает сам.

💎 К каким долгосрочным последствиям могут приводить принятые им проектные решения.

💎 Какие архитектурные «странности» можно ожидать от платформ и какими историческими параллелями они вызваны.





🗺 ОБЗОР ВЫПОЛНЕННЫХ РАБОТ (ROADMAP)



📑 Лабораторная работа №1: Технический семинар (Доклад)

👉 Тема: Rowhammer: Аппаратная уязвимость DRAM и методы архитектурной защиты.

👉 Результат: Проведен глубокий физический и архитектурный анализ механизмов нарушения изоляции ячеек динамической памяти (bit-flipping), изучены методы атак (TRR, TRRespass) и оценены способы программно-аппаратной защиты вычислительных систем.



📑 Лабораторная работа №2: Аналитические эссе

👉 Задача: Написание критических рецензий объемом 1024-4096 символов в формате Double-Blind (двойное слепое анонимное peer-review).

👉 Эссе №1: The Future of Programming (Bret Victor, 2013). Анализ четырех упущенных индустрией векторов развития и их пересечения с современными ИИ-ассистентами.

👉 Эссе №2: "Stop Writing Dead Programs" (Jack Rusher, 2022). Критика разорванного цикла эпохи перфокарт, анализ концепций "живых" систем и гибридного будущего продакшена.



📑 Лабораторная работа №3: Опыты (Моделирование 5 архитектур)

👉 Задача: Программная реализация алгоритмов на низкоуровневом ассемблере симулятора Wrench со строгим соблюдением спецификаций ввода-вывода (ячейки 0x80/0x84), обработки ошибок (-1) и переполнения (0xCC).

👉 Архитектура acc32: Алгоритм reverse\_string\_pstr (Реверс строки Pascal).

👉 Архитектура f32a: Алгоритм little\_to\_big\_endian (Изменение порядка байт).

👉 Архитектура m68k: Алгоритм stack\_based\_calculator (Стековый калькулятор RPN).

👉 Архитектура risc-iv-32: Алгоритм fibonacci (Числа Фибоначчи).

👉 Кастомная архитектура: Проектирование архитектуры m68k.



📑 Cхема архитектуры m68k

👉 Результат: Разработана детальная графическая принципиальная схема для CISC-архитектуры m68k на уровне логических узлов. Отображена структура регистрового файла (D0-D7, A0-A7), АЛУ с входными мультиплексорами (MUX), блок Branch Control и дешифратор адреса для Memory-Mapped I/O (MMIO).



📑 Финальный контроль: Экзамен

👉 Результат: Успешно пройден итоговый теоретический экзаменационный контроль по материалам курса.
