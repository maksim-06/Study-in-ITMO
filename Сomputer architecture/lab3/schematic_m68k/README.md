📐 ОПИСАНИЕ ПРИНЦИПИАЛЬНОЙ СХЕМЫ ПРОЦЕССОРА (M68K ISA)



Разработанная графическая схема представляет собой функциональную структуру CISC-процессора, адаптированную под систему команд m68k и поддерживающую расширенные режимы адресации оперативной памяти.



📌 ОСНОВНЫЕ АРХИТЕКТУРНЫЕ БЛОКИ СХЕМЫ



👉 Регистровый файл (Register File):

&#x20;  - Содержит 8 универсальных регистров данных (D0–D7) для выполнения арифметико-логических операций.

&#x20;  - Содержит 8 адресных регистров (A0–A7) для хранения указателей и организации работы со стеком (включая указатель стека). На входе регистровой группы установлен мультиплексор (MUX) выбора активного регистра.



👉 Арифметико-логическое устройство (АЛУ / ALU):

&#x20;  - Оснащено входными мультиплексорами (MUX) для выбора источников операндов. В качестве операндов могут выступать данные из регистров, регистр данных (DR), программный счетчик (PC) или регистр инструкций (IR).

&#x20;  - На выходе АЛУ формируются шина данных alu\_out и регистр флагов состояния (flags), отвечающий за условные переходы.



👉 Узел управления и ветвления (Control Logic):

&#x20;  - Блок управления ветвлением (Branch Control Unit): анализирует текущие флаги состояния АЛУ и код операции из регистра инструкций (IR) для фиксации переходов.

&#x20;  - Главное устройство управления (Control Unit): координирует работу всех узлов процессора, генерируя внутренние управляющие сигналы (control signals) на основе текущей инструкции.



👉 Подсистема памяти и ввода-вывода (Memory \& MMIO):

&#x20;  - Дешифратор адреса (Address Code Decoder): осуществляет Memory-Mapped I/O (MMIO). Он разделяет обращения к основной памяти (Memory) и к портам ввода-вывода (IO IN / IO OUT), управляя сигналами чтения/записи (mem\_wr, mem\_r).

&#x20;  - Входной мультиплексор памяти (MUX): коммутирует потоки данных, поступающие в процессор из оперативной памяти или внешнего порта ввода (IO IN), перенаправляя их в регистр данных (DR) или регистр инструкций (IR).
