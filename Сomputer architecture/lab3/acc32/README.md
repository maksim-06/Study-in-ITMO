💎 АЛГОРИТМ: РЕВЕРС СТРОКИ PASCAL (REVERSE\_STRING\_PSTR) — ARCH: ACC32



В этом модуле представлена низкоуровневая реализация алгоритма инвертирования строки формата Pascal String, адаптированная под систему команд и жесткие ограничения аккумуляторного процессора acc32.



📌 ТЕХНИЧЕСКОЕ ЗАДАНИЕ



👉 Суть задачи: Считать входную строку, завершающуюся символом новой строки (\\n), развернуть её в обратном порядке и сохранить результат в памяти в формате Pascal String (где первый байт хранит текущую длину строки).

👉 Ограничение буфера: Фиксированный размер буфера под сообщение составляет 32 байта (0x20) строго с адреса 0x00 (ячейки mem\[0..31]).

👉 Инициализация: Незаполненные символами ячейки буфера по умолчанию заполняются символом подчеркивания "\_".

👉 Обработка переполнения: Если входная строка превышает размер выделенного буфера, алгоритм корректно обрабатывает ошибку (выход за область определения).



📥 ВХОДНЫЕ И ВЫХОДНЫЕ ДАННЫЕ (ПРИМЕРЫ ВАЛИДАЦИИ)



🔹 Тест №1:

&#x20;  Ввод: 'hello\\n'

&#x20;  Выход: 'olleh'

&#x20;  Состояние памяти mem\[0..31] в шестнадцатеричном виде (hex):

&#x20;  05 6f 6c 6c 65 68 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f

&#x20;  (где 05 — длина строки, далее ASCII-коды букв 'o', 'l', 'l', 'e', 'h', а затем заглушки 5f, то есть '\_')



🔹 Тест №2:

&#x20;  Ввод: 'world!\\n'

&#x20;  Выход: '!dlrow'

&#x20;  Состояние памяти mem\[0..31] в шестнадцатеричном виде (hex):

&#x20;  06 21 64 6c 72 6f 77 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f 5f



🛠 ОСОБЕННОСТИ РЕАЛИЗАЦИИ НА ACC32



🔥 Логика работы с аккумулятором: Так как в архитектуре acc32 все базовые операции (загрузка, сравнение, сохранение) завязаны на единственный регистр-аккумулятор, алгоритм посимвольного реверса реализован через косвенную адресацию и инкремент/декремент адресных указателей прямо в ячейках памяти.

🔥 Преобразование формата: Логика чтения динамически подсчитывает количество символов до '\\n', записывает итоговую длину в нулевой байт (0x00), а затем выполняет инверсию элементов внутри ограниченного окна памяти.



🔗 Ссылка на отчет автотестирования ЛР: https://wrench.edu.swampbuds.me/report/1aa8ecef-e9aa-4001-ba1b-94dee18e7298



Решение:

.data

.org 0x100





i:      .word 50 ; индекс по которому вытаскиваем букву

nl:     .word 0x0A ; маркер конца входной троки

one:    .word 1

cc:     .word 0xCC ; если рез не может быть корректно посчитан вернуть cc, cc, ..., cc

len:    .word 0

adr:    .word 0 ; индекс записи

buf:    .word 0x1F ; максимальный индекс для памяти 

val50: .word 50



tmp:    .word 0 ; временная переменная

j:    	.word 0x50 ; временный индекс 

input\_addr: .word 0x80

output\_addr: .word 0x84

mask: .word 0xFF





.text

&#x20;   .org 0x200 



\_start:

&#x20;   load input\_addr

&#x20;   load\_acc



&#x09;store\_addr tmp ; сохраняем каждую букву отдельно из порта 



&#x20;   load\_addr j

&#x20;   store\_addr adr



&#x20;   load\_addr tmp

&#x20;   store\_ind adr ; сохраняем букву в tmp по адресу adr (который j)



&#x20;   load\_addr tmp

&#x09;sub nl

&#x09;beqz reverse ; проверяем что буква не \\n



&#x09;load\_addr j

&#x09;add one ; увеличиваем индекс j

&#x09;store\_addr j



&#x20;   load\_addr len

&#x20;   add one

&#x20;   store\_addr len ; считаем длину



&#x20;   load\_addr len

&#x20;   sub buf

&#x20;   bgtz write\_cc       ; перехват переполнения

&#x20;   jmp \_start    



reverse:

&#x20;   load\_imm 0

&#x20;   store\_addr adr

&#x20;   

&#x20;   load\_addr len

&#x20;   

&#x20;   beqz skip\_cycles

&#x20;   

&#x20;   store\_ind adr



&#x20;   load\_addr adr

&#x20;   add one

&#x20;   store\_addr adr



&#x09;load\_addr j

&#x09;sub one

&#x09;store\_addr i ; до этого момента индекс был на \\n, а должен быть на последней букве



&#x09;jmp reverse\_loop



skip\_cycles:

&#x20;   load\_imm 0

&#x20;   store\_addr tmp

&#x20;   load\_addr len

&#x20;   store\_ind tmp       ; mem\[0] = 0

&#x20;   

&#x20;   load\_imm 1

&#x20;   store\_addr i        ; i = 1 (начало заполнения '\_')

&#x20;   jmp end\_loop

&#x09;



reverse\_loop:

&#x09;load\_addr i ; загружаем индекс i (место последней буквы слова, пример 5, 4, 3, …)

&#x20;   load\_acc



&#x09;store\_ind adr ; индекс записи в первый раз равен 1 (после длины слова)	



&#x20;   load\_addr adr

&#x09;add one

&#x09;store\_addr adr ; увеличиваем +1 индекс записи

&#x09;

&#x20;   load\_addr i

&#x20;   sub val50

&#x20;   beqz end ; Если аккумулятор равен нулю, переключитесь на конкретный адрес.	



&#x20;   load\_addr i

&#x20;   sub one

&#x20;   store\_addr i

&#x20;   

&#x20;   

&#x09;

&#x09;jmp reverse\_loop



end: ; длина строки лежит и строка лежит перевернутая в памяти, переходим к заполнению \_

&#x20;   

&#x09;load\_addr len

&#x09;add one

&#x09;store\_addr i ; к длине +1 (индекс после последней буквы)

&#x09;

&#x09;jmp end\_loop

&#x09;

end\_loop:

&#x09;

&#x20;   load\_addr i

&#x20;   sub buf

&#x20;   bgtz check\_len      ; i > 31? Буфер полон → выход



&#x20;   load\_addr i

&#x20;   store\_addr tmp

&#x20;   load\_imm 0x5F

&#x20;   store\_ind tmp



&#x20;   load\_addr i

&#x20;   add one

&#x20;   store\_addr i

&#x20;   jmp end\_loop



check\_len:              

&#x20;   load\_addr len

&#x20;   beqz go\_out         ; len=0? Строка пустая → сразу выход, порт останется \[]

&#x20;   jmp save\_port



save\_port: ; сохраняем ответ с индекса 1 в порт 0x84

&#x20;   load\_imm 1

&#x20;   store\_addr adr

&#x20;   

&#x20;   jmp save\_port\_loop



save\_port\_loop:

&#x20;   load\_addr adr

&#x20;   

&#x20;   load\_acc ; загружаем букву по индексу (1, 2, 3, ...)

&#x20;   and mask

&#x20;   store\_ind output\_addr ; загружаем в порт



&#x20;   load\_addr adr

&#x20;   sub len

&#x20;   beqz go\_out ; проверяем, дошли ли до последнего индекса 

; (в примере с hello это длина = 5, то есть индекс после буквы 5)



&#x20;   load\_addr adr

&#x20;   add one

&#x20;   store\_addr adr

&#x20;   

&#x20;   jmp save\_port\_loop



write\_cc:

&#x20;   load\_imm -858993460    ; 0xCCCCCCCC в десятичном виде

&#x20;   store\_addr 0x84        ; Записываем 32-битный код ошибки в порт вывода



go\_out:

&#x09;halt
