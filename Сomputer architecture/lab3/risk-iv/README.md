💎 АЛГОРИТМ: ЧИСЛА ФИБОНАЧЧИ (FIBONACCI) — ARCH: RISC-IV-32



В этом модуле представлена низкоуровневая реализация алгоритма вычисления N-го числа Фибоначчи, разработанная под архитектуру risc-iv.



📌 ТЕХНИЧЕСКОЕ ЗАДАНИЕ И ОГРАНИЧЕНИЯ

👉 Суть задачи: Считать из входного порта 0x80 порядковый номер числа N. Рассчитать значение соответствующего члена последовательности Фибоначчи и записать итоговый ответ в порт вывода 0x84.

👉 Область определения: 

&#x20;  - Значения N = 0 и N = 1 возвращают 0 и 1 соответственно.

&#x20;  - Отрицательные значения входного аргумента (N < 0) являются некорректными, при их передаче алгоритм возвращает -1.

&#x20;  - При превышении разрядной сетки знакового 32-битного машинного слова (переполнение) возвращается маркер ошибки 0xCCCCCCCC.



🛠 ОСОБЕННОСТИ РЕАЛИЗАЦИИ НА RISC-IV-32

🔥 Процедурный стек и рекурсия: В соответствии с жесткими требованиями к данной ISA, архитектура программы построена на базе вложенных вызовов подпрограмм с явным сохранением контекста регистров (регистр адреса возврата и временные регистры) в системном стеке памяти. Вычисление реализовано с использованием итерационного спуска, демонстрирующего работу указателя стека.

🔥 Ортогональность команд: Код задействует фиксированный формат инструкций, базовые арифметические команды и условные переходы RISC-процессора для организации счетчиков циклов и проверок базовых случаев (N=0, N=1).



🤖 АВТОТЕСТИРОВАНИЕ

🔗 Ссылка на отчет автотестирования: https://wrench.edu.swampbuds.me/report/2ff6171b-284b-4954-a130-27960cd12ab4



Решение:

.data

adr\_input:     .word 0x80

adr\_output:    .word 0x84

overflow:    .word 0xCCCCCCCC





.text

\_start:

&#x20;   .org 0x100

&#x20;   lui sp, 0x1



&#x20;   lui  t0, %hi(adr\_input)

&#x20;   addi t0, t0, %lo(adr\_input)

&#x20;   lw   t0, 0(t0)

&#x20;   lw   a0, 0(t0)



&#x20;   jal ra, fib\_compute

&#x20;   

&#x20;   lui  t0, %hi(adr\_output)

&#x20;   addi t0, t0, %lo(adr\_output)

&#x20;   lw   t0, 0(t0)

&#x20;   sw   a0, 0(t0)

&#x20;   

&#x20;   

&#x20;   halt



fib\_compute:

&#x20;   addi sp, sp, -8

&#x20;   sw   ra, 4(sp)

&#x20;   sw   s0, 0(sp)

&#x20;   mv   s0, a0



&#x20;   slti t4, s0, 0

&#x20;   beqz t4, check\_zero       ; если >= 0, идём дальше

&#x20;   

&#x20;   jal  ra, ret\_neg



&#x20;   j    epilogue             ; сразу выход

check\_zero:

&#x20;   bnez s0, after\_zero\_check

&#x20;   jal  ra, ret\_zero 

&#x20;   j    epilogue



after\_zero\_check:

&#x20;   addi t0, zero, 2

&#x20;   addi t1, zero, 0

&#x20;   addi t2, zero, 1



fib\_loop:

&#x20;   bgt  t0, s0, fib\_end

&#x20;   add  t3, t1, t2



&#x20;   slti t5, t3, 0

&#x20;   beqz t5, skip\_overflow    ; если >= 0, пропускаем

&#x20;   

&#x20;   lui  t1, %hi(overflow)

&#x20;   addi t1, t1, %lo(overflow)

&#x20;   lw   a0, 0(t1)

&#x20;   j    epilogue    



skip\_overflow:

&#x20;   mv   t1, t2

&#x20;   mv   t2, t3

&#x20;   addi t0, t0, 1

&#x20;   j    fib\_loop



ret\_zero:

&#x20;   addi a0, zero, 0

&#x20;   jr ra

ret\_neg:

&#x20;   addi a0, zero, -1

&#x20;   jr ra



fib\_end:

&#x20;   mv   a0, t2



epilogue:

&#x20;   lw   ra, 4(sp)

&#x20;   lw   s0, 0(sp)

&#x20;   addi sp, sp, 8

&#x20;   jr   ra
