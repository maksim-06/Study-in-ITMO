💎 АЛГОРИТМ: КОНВЕРТАЦИЯ БАЙТ-ОРДЕРА (LITTLE\_TO\_BIG\_ENDIAN) — ARCH: F32A



В этом модуле представлена низкоуровневая реализация алгоритма изменения порядка байт в 32-битном машинном слове, разработанная под стековую архитектуру f32a.



📌 ТЕХНИЧЕСКОЕ ЗАДАНИЕ И ОГРАНИЧЕНИЯ

👉 Суть задачи: Считать 32-битное целое число из входного порта 0x80, изменить порядок следования его байт на противоположный (перевод из формата Little-Endian в Big-Endian) и отправить результат в порт 0x84.

👉 Пример валидации: На входе число 305419896 (0x12345678) преобразуется в 2018915346 (0x78563412).



🛠 ОСОБЕННОСТИ РЕАЛИЗАЦИИ НА F32A

🔥 Процедурная абстракция: Согласно требованиям ISA, вся логика разбита на изолированные подпрограммы. Выделены отдельные процедуры для логических сдвигов влево и вправо (shl8, shl16, shl24, shr8, shr16, shr24), реализованные через базовые команды умножения (2\*) и деления (2/).

🔥 Двухстековая организация: Для побайтовой разборки числа применяется маскирование (0xFF и and). Выделенные байты последовательно переносятся во временный стек возвратов с помощью команды (>r), а затем извлекаются обратно через (r>) для сборки итогового слова в обратном порядке при помощи сложения (+). Применена понятная структура именования меток.



🤖 АВТОТЕСТИРОВАНИЕ

🔗 Ссылка на отчет автотестирования: https://wrench.edu.swampbuds.me/report/225e6e53-69a2-4cd0-9d87-2281524a649b



Решение:

.text



&#x20;   .org 0x100



\_start:

&#x20;   @p 0x80               

&#x20;   little\_to\_big\_endian   

&#x20;   !p 0x84                

&#x20;   halt





little\_to\_big\_endian:

&#x20;   dup 0xFF and >r        

&#x20;   dup shr8 0xFF and >r   

&#x20;   dup shr16 0xFF and >r   

&#x20;   shr24 0xFF and >r        

&#x20;   

&#x20;   r>             

&#x20;   r> shl8  +        

&#x20;   r> shl16 +       

&#x20;   r> shl24 +     

&#x20;   ;

shl8:  2\* 2\* 2\* 2\* 2\* 2\* 2\* 2\* ;

shl16: shl8 shl8 ;

shl24: shl16 shl8 ;



shr8:  2/ 2/ 2/ 2/ 2/ 2/ 2/ 2/ ;

shr16: shr8 shr8 ;

shr24: shr16 shr8 ;
